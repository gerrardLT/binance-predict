# BTC 5min AI 自动预测系统需求文档

## 1. 项目名称

**BTC 5min AI 全自动预测系统**

---

## 2. 项目定位

本项目用于构建一个由 AI 完全接管预测判断的 BTC 短周期预测系统。

系统不只是调用一个机器学习模型输出涨跌概率，而是让 AI 像一个“自动化交易分析员”一样，持续观察市场数据、理解行情结构、判断未来 5 分钟 BTC 价格方向，并输出结构化预测结果。

本项目第一阶段只做：

```text
BTC/USDT
5分钟预测
AI自动判断
预测结果记录
预测复盘
模型自我优化
```

暂不直接自动下单。

---

## 3. 核心目标

系统最终需要实现：

```text
AI 自动读取市场数据
AI 自动判断当前市场状态
AI 自动预测 BTC 未来 5 分钟方向
AI 自动输出上涨概率、下跌概率、置信度
AI 自动解释判断原因
AI 自动记录每一次预测
AI 自动在 5 分钟后验证预测结果
AI 自动总结错误原因
AI 自动优化后续判断规则
```

系统输出不是一句“看涨/看跌”，而是完整预测结论：

```json
{
  "symbol": "BTCUSDT",
  "horizon": "5m",
  "prediction": "UP",
  "p_up": 0.64,
  "p_down": 0.36,
  "confidence": 0.72,
  "market_state": "short_term_momentum_up",
  "risk_level": "medium",
  "reason": [
    "1分钟和3分钟动量同步转强",
    "主动买入量明显高于主动卖出量",
    "盘口买盘深度占优",
    "合约持仓量上升，价格同步上涨"
  ],
  "avoid_reason": null
}
```

---

## 4. 一句话定义

本项目不是“写一个指标策略”，而是构建一个：

```text
由 AI Agent 主导的 BTC 5分钟市场判断系统
```

AI 的职责是：

```text
看数据
理解市场
做判断
给概率
给理由
做复盘
自我修正
```

---

## 5. 项目边界

## 5.1 本项目只做

```text
BTCUSDT 价格预测
5分钟周期
上涨/下跌/不确定判断
AI Agent 自动分析
多数据源融合
预测日志
结果验证
错误复盘
策略迭代
```

---

## 5.2 本项目暂不做

```text
其他币种预测
长周期投资判断
体育预测
新闻预测市场
自动梭哈交易
高杠杆交易
马丁格尔倍投
无风控自动交易
```

---

## 5.3 第一阶段默认模式

```text
SIGNAL_ONLY
```

即：

```text
AI 只负责预测和解释
不直接控制资金
不直接下单
不直接开合约
```

后续可扩展为：

```text
MANUAL_CONFIRM：AI 给信号，人工确认后交易
AUTO_TRADE：AI 在风控边界内自动交易
```

# 6. AI 完全接管预测的核心设计

## 6.1 AI 不是单一大模型

这里的 AI 不应该只是一个 LLM。

正确结构应该是：

```text
数据采集系统
+
特征计算系统
+
机器学习预测模型
+
规则风控系统
+
LLM 行情分析 Agent
+
复盘优化 Agent
```

也就是说，AI 判断由多个部分共同完成。

---

## 6.2 AI Agent 角色划分

系统内部建议拆成 7 个 AI Agent。

---

### Agent 1：行情观察员

职责：

```text
读取 BTC 当前价格
读取 1m / 5m K线
读取短周期涨跌幅
读取成交量变化
读取盘口变化
判断当前市场是否活跃
```

输出：

```json
{
  "price_state": "rising",
  "momentum": "strong",
  "volatility": "medium",
  "volume_state": "expanding",
  "summary": "BTC 当前短周期价格处于上行动量中"
}
```

---

### Agent 2：盘口分析员

职责：

```text
分析 best bid / best ask
分析买卖盘深度
分析 spread
分析盘口倾斜方向
分析盘口是否异常
判断当前买盘强还是卖盘强
```

输出：

```json
{
  "orderbook_bias": "buy_side_stronger",
  "spread_state": "normal",
  "liquidity_state": "good",
  "imbalance": 0.63,
  "summary": "盘口买盘深度明显高于卖盘"
}
```

币安现货行情接口支持深度信息、近期成交、聚合成交、K线等市场数据，其中 K线接口支持 `1m、3m、5m、15m、30m` 等周期，适合本系统采集 BTCUSDT 5min 预测所需的基础行情。

---

### Agent 3：成交流分析员

职责：

```text
分析主动买入量
分析主动卖出量
分析大单成交
```

输出：

```json

  "trade_speed": "increasing",

}
```

---

职责：


```json

  "funding_pressure": "neutral",
  "long_short_ratio": "long_increasing",
```

职责：


# 7. 预测逻辑设计

## 7.1 AI 最终判断不等于模型输出

机器学习模型只提供概率。

最终预测由 AI 总决策官决定。

例如：

```text
模型输出 p_up = 0.62
但盘口卖压明显增强
主动卖出量突然放大
合约持仓量上升但价格不上涨
```

AI 可以否决模型，输出：

```text
NO_TRADE 或 DOWN
```

---

## 7.2 AI 必须支持三种预测结果

```text
UP：预测未来5分钟上涨
DOWN：预测未来5分钟下跌
NO_TRADE：市场不清晰，不做方向判断
```

注意：

```text
NO_TRADE 是重要结果，不是失败。
```

很多时候不预测，比乱预测更重要。

---

## 7.3 AI 判断层级

AI 判断必须分 5 层。

---

### 第一层：市场是否可预测

判断：

```text
当前行情是否活跃
成交量是否足够
盘口是否正常
spread 是否过大
是否存在异常跳价
是否临近重大宏观事件
是否出现 API 延迟
```

如果不可预测，直接输出：

```text
NO_TRADE
```

---

### 第二层：短周期方向

判断：

```text
过去10秒方向
过去30秒方向
过去1分钟方向
过去3分钟方向
过去5分钟方向
```

---

### 第三层：资金推动

判断：

```text
上涨是否有主动买入推动
下跌是否有主动卖出推动
价格变化是否只是低流动性噪声
```

---

### 第四层：合约确认

判断：

```text
现货上涨，合约是否同步上涨
价格上涨时 OI 是否增加
价格下跌时 OI 是否增加
多空比是否极端
资金费率是否过热
```

---

### 第五层：最终预测

AI 综合输出：

```text
UP
DOWN
NO_TRADE
```

并给出：

```text
上涨概率
下跌概率
置信度
风险等级
判断理由
反向风险
```

# 8. AI 决策提示词需求

## 8.1 AI 总决策官系统提示词

```text
你是 BTC 5分钟短周期预测系统的 AI 总决策官。

你的任务不是聊天，而是根据结构化市场数据，判断 BTCUSDT 在未来 5 分钟内更可能上涨、下跌，还是不适合预测。

你必须遵守以下原则：

1. 只根据输入数据判断，不得凭空编造行情。
2. 如果数据矛盾，优先输出 NO_TRADE。
3. 如果置信度不足，优先输出 NO_TRADE。
4. 不追求每次都预测，宁可少预测，也不要乱预测。

10. 不允许输出投资建议，只输出预测信号和风险解释。

你的输出字段必须包含：
- final_prediction: UP / DOWN / NO_TRADE
- p_up
- p_down
- confidence
- risk_level
- market_state
- reason
- invalid_conditions
- reverse_risk
- required_followup_data
```

# 11. AI 判断规则

## 11.1 UP 判断条件

AI 可以判断 UP 的典型条件：

```text
短周期价格动量为正
1m 和 3m 方向一致
成交量放大
主动买入量占优
盘口买盘深度占优
合约持仓量上升
资金费率未明显过热
机器学习模型 p_up 高于阈值
```

---

## 11.2 DOWN 判断条件

AI 可以判断 DOWN 的典型条件：

```text
短周期价格动量为负
1m 和 3m 方向一致向下
成交量放大
主动卖出量占优
盘口卖盘深度占优
合约持仓量上升
多头拥挤或多头平仓迹象明显
机器学习模型 p_down 高于阈值
```

---

## 11.3 NO_TRADE 判断条件

以下情况必须优先 NO_TRADE：

```text
数据延迟
行情断流
盘口异常
spread 过大
成交量过低
价格来回震荡
模型之间分歧大
现货和合约信号冲突
AI 置信度不足
重大消息前后异常波动
连续预测错误后进入冷却期
```

---

# 12. AI 置信度设计

AI 置信度不能凭空生成。

置信度由以下因素组成：

```text
模型概率强度
多个模型是否一致
价格方向是否一致
成交量是否确认
盘口是否确认
主动买卖是否确认
合约数据是否确认
历史相似场景胜率
当前市场是否异常
```

建议初始规则：

```text
confidence >= 0.75：强信号
0.60 <= confidence < 0.75：普通信号
0.50 <= confidence < 0.60：弱信号
confidence < 0.50：NO_TRADE
```


## 13.2 标签定义

```text
如果 price(T+5min) > price(T)，实际结果 = UP
如果 price(T+5min) < price(T)，实际结果 = DOWN
如果 price(T+5min) ≈ price(T)，实际结果 = FLAT
```

---

## 13.3 更推荐的交易型标签

因为 5 分钟波动很小，必须考虑噪声：

```text
future_return = price(T+5min) / price(T) - 1

如果 future_return > 0.05%，actual_label = UP
如果 future_return < -0.05%，actual_label = DOWN
否则 actual_label = NOISE
```

阈值可配置：

```text
noise_threshold = 0.03% - 0.08%
```

```text
自动判断
自动给概率
自动解释
自动复盘
自动修正
```

系统每次输出都应该回答 6 个问题：

```text
1. 未来5分钟 BTC 更可能涨还是跌？
2. 上涨概率是多少？
3. 下跌概率是多少？
4. 置信度是多少？
5. 为什么这么判断？
6. 什么情况下这个判断会失效？
```

---

# 28. 核心原则

```text
AI 接管预测，不等于 AI 盲目预测
AI 必须基于数据判断
数据矛盾时不预测
置信度不足时不预测
复盘比单次预测更重要
NO_TRADE 是高质量判断的一部分
预测系统先验证，再交易
```

# 29. 最小闭环

第一版只要完成这个闭环即可：

```text
采集 BTC 数据

模型输出概率
AI Agent 综合判断
输出 5min 预测
5分钟后验证结果
AI 自动复盘
统计表现
```

这就是本项目的最小可行版本。
